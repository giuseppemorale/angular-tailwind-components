# Icons

Le icone sono **[Solar Line Duotone](https://solar-icons.vercel.app/icons)** (480 Design), licenza **CC BY 4.0**. Gli SVG master sono **64×64** nel viewBox del set Solar (`24×24` tipico); il componente imposta larghezza/altezza in pixel tramite **`size`** (16–64).

## Uso in TypeScript

Importa il tipo e l’array dei nomi dalla libreria:

```typescript
import type { TailwindSolarIcon } from 'angular-tailwind-components';
import { TAILWIND_SOLAR_ICON_NAMES } from 'angular-tailwind-components';

const icon: TailwindSolarIcon = 'bell';
```

Per ogni icona la **stringa da usare** è il nome in **kebab-case** tra apici singoli (come in `TailwindSolarIcon`), ad esempio `'bell'` oppure `'home-smile'`.

## Componente `tailwind-icon`

Per mostrare un’icona usa **`TailwindIcon`** con:

- **`icon`**: `TailwindSolarIcon` (obbligatorio)
- **`size`**: numero **pixel** per larghezza e altezza (**16–64**, valori fuori range vengono clampati); se omesso usa il token **`TAILWIND_ICON_SIZE`** se presente, altrimenti **24**

Il componente imposta **`src`** su **`/tailwind-icons/<nome>.svg`** (URL dalla root del sito che serve l’app). Le SVG non sono incorporate nel bundle JS: il browser le scarica come risorse statiche.

```typescript
import { TailwindIcon } from 'angular-tailwind-components';

@Component({
  imports: [TailwindIcon],
  template: `<tailwind-icon icon="bell" [size]="16" />`
})
export class ExampleComponent {}
```

## Build della libreria e pacchetto npm

Prima di **`npm run build`** viene eseguito **`node scripts/copy-icons.js sync`**, che rigenera **`public/tailwind-icons/*.svg`** e **`models/icons.ts`** da `@iconify-json/solar`. Subito dopo la compilazione, **`node scripts/copy-icons.js dist`** copia gli SVG nel pacchetto pubblicato in **`tailwind-icons/`** (accanto a `fesm2022`, ecc.).

## App che usa la libreria (build di produzione)

Nel **`angular.json`** (target **`build`** dell’app) aggiungi gli asset così le richieste a **`/tailwind-icons/nome.svg`** risolvono correttamente:

```json
"assets": [
  {
    "glob": "**/*",
    "input": "node_modules/angular-tailwind-components/tailwind-icons",
    "output": "/tailwind-icons"
  }
]
```

Il path **`input`** è relativo alla root del workspace dell’app. In sviluppo **`ng serve`** copia gli stessi file nella cartella di output con prefisso **`/tailwind-icons/`**.

## Dimensione predefinita: `TAILWIND_ICON_SIZE`

Il token **`TAILWIND_ICON_SIZE`** (`InjectionToken<number>`) imposta il **default** di **`size`** su tutte le **`tailwind-icon`** dell’albero che ricevono quel provider. Se il token **non** è fornito, il default è **24** (px).

```typescript
import { bootstrapApplication } from '@angular/platform-browser';
import { TAILWIND_ICON_SIZE } from 'angular-tailwind-components';
import { AppComponent } from './app.component';

bootstrapApplication(AppComponent, {
  providers: [{ provide: TAILWIND_ICON_SIZE, useValue: 20 }]
});
```

Puoi anche limitarlo a una parte dell’app con **`providers`** su un componente contenitore:

```typescript
@Component({
  selector: 'app-settings-shell',
  imports: [TailwindIcon],
  providers: [{ provide: TAILWIND_ICON_SIZE, useValue: 20 }],
  template: `<tailwind-icon icon="settings" />`
})
export class SettingsShellComponent {}
```

Passare esplicitamente **`[size]`** sulla singola icona ha sempre la precedenza sul valore del token.

## Percorsi dei file SVG

Nella repo della libreria gli SVG sorgente stanno in **`public/tailwind-icons/`** — un file `.svg` per ogni nome in `TailwindSolarIcon`. Nel pacchetto npm sono nella cartella **`tailwind-icons/`**.
